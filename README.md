# LOST Database Project

This project demonstrates the design and implementation of a relational database using MySQL. The system models a tour management environment, including clients, guides, tours, locations, and scheduled outings.

## Key Features
- Designed normalized database schema with multiple related tables and foreign key relationships
- Implemented data population scripts to simulate real-world usage
- Developed SQL queries and views using joins, aggregations, and filtering
- Created reusable views to support reporting and data analysis

## Example Use Cases
- Identify clients who have not booked any outings
- Count number of clients scheduled for each outing
- Retrieve guides qualified for specific tours
- Filter outings based on date and assigned guides

## Technologies
- MySQL
- SQL (DDL, DML, JOINs, Views)

## Project Structure
- `schema/` – database structure and table creation
- `data/` – insert scripts to populate tables
- `queries/` – SQL queries and views
- `model/` – database design (MySQL Workbench)

## Notes
This project was built to strengthen understanding of relational database design, data modeling, and query development.
